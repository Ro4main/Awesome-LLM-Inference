# Weight/Activation Quantize/Compress 论文优先级排序

基于**影响力、创新性、工业界采用度**三个维度，将 35 篇论文分为三级。

---

## 一、必读（8 篇）— 奠基性 / 工业级标配 / 开辟新方向

| 序号 | 论文 | 时间 | 机构 | 理由 |
|:---:|------|:---:|------|------|
| 1 | **LLM.int8()** | 2022.08 | Facebook AI Research 等 | 首次提出离群值分离机制，量化入门必读，bitsandbytes 库被广泛使用 |
| 2 | **SmoothQuant** | 2022.11 | MIT 等 | 激活量化的里程碑，"难度迁移"思想极其巧妙，W8A8 的标杆 |
| 3 | **GPTQ** | 2022.10 | IST Austria 等 | 4-bit PTQ 的标准范式，几乎所有推理框架都支持，理论基础扎实 |
| 4 | **AWQ** | 2023.06 | MIT 等 | 激活感知权重量化，精度高、速度快，当前工业界最主流的 W4 方案 |
| 5 | **ZeroQuant** | 2022.06 | Microsoft | 微软系列工作的起点，per-token/per-channel 粒度量化的系统性框架 |
| 6 | **QServe (W4A8KV4)** | 2024.05 | MIT & NVIDIA | 全栈量化系统设计的代表，算法+kernel+调度协同，工业级落地标杆 |
| 7 | **BitNet v2** | 2025.04 | Microsoft | 原生 1-bit 训练 + Hadamard 变换，极低比特的终极方向，2025 最新前沿 |
| 8 | **GuidedQuant** | 2025.05 | SNU & Samsung AI & Google | 端到端损失引导量化，突破了"逐层最小化误差"的局限，代表 PTQ 新范式 |

### 核心阅读路径

```
LLM.int8() → SmoothQuant → GPTQ → AWQ → QServe
  (理解离群点)  (激活量化)  (4-bit PTQ)  (更优 PTQ)  (全栈系统)
                                                          ↓
                                               BitNet v2 / GuidedQuant
                                               (原生低比特) (端到端引导)
```

### 补充篇主题阅读路线

#### 路线 1：浮点数量化深入（5 篇）

适合人群：想深入理解 FP8/FP6/FP4 等低精度浮点格式，关注硬件原生支持的量化方向。

```
阅读顺序：
  FP8-Quantization → FP8-LM → ZeroQuant-FP → LLM-FP4 → FP6-LLM

路线特点：
  从标准 FP8 格式的理论分析入手
  → FP8 训练与推理的系统实践
  → 再到更激进的 FP4 和非标准 FP6
  → 完整覆盖浮点数量化的演进脉络
```

| 顺序 | 论文 | 核心看点 |
|:---:|------|---------|
| 1 | **FP8-Quantization (2022.08)** | FP8 格式的理论基础，理解 E4M3 vs E5M2 两种格式的区别和适用场景 |
| 2 | **FP8-LM (2023.10)** | 微软用 FP8 训练 LLM 的系统实践，MS-AMP 库，训练-推理全链路 FP8 |
| 3 | **ZeroQuant-FP (2023.07)** | 首次提出 W4A8 浮点量化，权重 4-bit FP + 激活 8-bit，精度优于整数 |
| 4 | **LLM-FP4 (2023.10)** | 4-bit 浮点数量化，Meta & UST，FP4 格式在 LLM 上的全面评估 |
| 5 | **FP6-LLM (2024.01)** | 6-bit 浮点的算法+系统协同设计，非标准位宽如何高效实现，全栈优化的代表 |

#### 路线 2：稀疏与向量量化（3 篇）

适合人群：对"超越均匀量化"的更激进压缩方法感兴趣，关注稀疏化、向量量化等方向。

```
阅读顺序：
  SpQR → SqueezeLLM → VPTQ

路线特点：
  从"稀疏+量化"混合的 SpQR 开始
  → 稠密+稀疏分块的 SqueezeLLM
  → 更激进的向量量化 VPTQ
  → 探索"比 4-bit 更低"的极限压缩路径
```

| 顺序 | 论文 | 核心看点 |
|:---:|------|---------|
| 1 | **SpQR (2023.06)** | 稀疏量化表示，大部分权重 4-bit + 少数异常值高精度存储，接近无损压缩，技术思路独特 |
| 2 | **SqueezeLLM (2023.06)** | UC Berkeley，稠密+稀疏混合量化，敏感列用高精度，非敏感列用低精度 |
| 3 | **VPTQ (2024.09)** | 微软出品，向量 PTQ，用向量量化代替标量量化，极低比特（2-3 bit）的探索 |

#### 路线 3：1-bit 路线演进（2 篇 + 必读的 BitNet v2）

适合人群：对极低比特、原生量化训练感兴趣，想了解 1-bit LLM 的完整发展脉络。

```
阅读顺序：
  1-bit LLMs → BitNet a4.8 → BitNet v2（必读篇）

路线特点：
  从"1-bit LLM 是否需要完整 matmul"的理论分析切入
  → BitNet a4.8 的 4-bit 激活+稀疏化实践
  → BitNet v2 的 Hadamard 变换原生 4-bit 激活
  → 完整呈现 1-bit 模型从理论到工程的演进
```

| 顺序 | 论文 | 核心看点 |
|:---:|------|---------|
| 1 | **1-bit LLMs (2024.08)** | 理论分析：1-bit LLM 是否真的需要矩阵乘法？对 BitLinear 的计算本质有深刻洞察 |
| 2 | **BitNet a4.8 (2024.11)** | BitNet v2 的前置工作，4-bit 输入激活 + 8-bit 稀疏中间态，理解激活量化难点 |
| 3 | **BitNet v2 (2025.04)** | （必读篇）Hadamard 变换消除异常值，原生 4-bit 稠密激活，1-bit 研究的最新前沿 |

#### 路线 4：系统工程与工业实践（3 篇）

适合人群：关注工业部署、系统优化、工程实现，想了解大厂如何落地量化技术。

```
阅读顺序：
  WINT8/4 → ZeroQuant-V2 → ABQ-LLM

路线特点：
  从 NVIDIA & Microsoft 的 MoE 工业部署实践出发
  → 微软 ZeroQuant 系列的完善版（低秩补偿 LoRC）
  → 字节跳动的任意位宽量化加速系统
  → 聚焦"从算法到系统"的工程落地视角
```

| 顺序 | 论文 | 核心看点 |
|:---:|------|---------|
| 1 | **WINT8/4 (2022.11)** | NVIDIA & Microsoft 联合，MoE 模型的工业级量化部署，大规模生产实践经验 |
| 2 | **ZeroQuant-V2 (2023.03)** | 低秩补偿（LoRC）技术，用低秩矩阵补偿量化误差，系统性改进 PTQ |
| 3 | **ABQ-LLM (2024.08)** | 字节跳动出品，任意比特位宽的量化加速系统，系统实现和 kernel 优化有参考价值 |

#### 路线 5：新颖思路探索（2 篇 + SparQ）

适合人群：想拓展视野，了解量化领域的非主流但有启发性的创新思路。

```
阅读顺序：
  SpinQuant → SparQ Attention → （必读的 BitNet v2 Hadamard 变换）

路线特点：
  从"学习旋转矩阵来改善量化"的 SpinQuant
  → "用 Query 稀疏近似减少 KV 带宽"的 SparQ Attention
  → BitNet v2 的 Hadamard 正交变换
  → 共同主题：通过线性变换改变数据分布，从而改善量化
```

| 顺序 | 论文 | 核心看点 |
|:---:|------|---------|
| 1 | **SpinQuant (2024.05)** | Meta 出品，学习旋转矩阵 R，让 W' = R W 更适合量化，思路新颖优雅 |
| 2 | **SparQ Attention (2023.12)** | Graphcore，Query 侧稀疏化 + Top-K Key 选择，从带宽角度优化注意力 |

#### 时间有限推荐（3-5 篇）

从每个方向选最有代表性的：

| 推荐 | 论文 | 理由 |
|:---:|------|------|

| 3 | **ZeroQuant-V2** | LoRC 误差补偿技术有通用性 |

#### 按兴趣深入组合

- **硬件/系统向**：路线 1（FP 浮点）+ 路线 4（系统实践）
